# Phase 12X action-contract inventory

This inventory describes the checked-in Phase 12A–12J client/backend contract. It is an engineering artifact, not evidence that the functions are deployed. Every function below exists locally and is expected in the release environment. A missing function is mapped to `feature_not_deployed`; a relay/availability failure is mapped to `service_unavailable`; an action outside the authoritative contract is mapped to `unsupported_action`. The UI shows scoped copy and never substitutes an authoritative empty result.

## Integrity notes

- Swift owns `SDOrgAdminAction`; TypeScript owns `ORG_ADMIN_ACTIONS`. Synchronization tests compare every Swift raw value with the backend list, and backend tests prove every listed value has a dispatch branch.
- `create_season` is the exact client action and exact checked-in backend action. A backend returning `unknown_action` for it is older than this local contract.
- The `org_admin` backend now rejects unlisted actions as `unsupported_action`, logs the internal value server-side, and never returns `Unknown Action` copy.
- Season mutation retry uses the same request ID until success. The backend replays an audited prior result for that actor/organization/action/request ID, avoiding a duplicate on a sequential retry after a lost response.
- The client classifier retains technical diagnostics in debug logs while user presentation is limited to controlled categories.
